Getting Started
Analytics Track — Snowflake Setup

Three steps to get your Analytics track environment running. Complete this before writing any code.

1
Create your database and schema
Log in to your Snowflake account. Open a new Worksheet and run the commands below to set up your hackathon environment.
CREATE DATABASE IF NOT EXISTS HACKATHON;
CREATE SCHEMA IF NOT EXISTS HACKATHON.DATA;
2
Load your CSV file
In the left sidebar, go to Data → Add Data → Load data into a Table. Upload your CSV, select database HACKATHON and schema DATA. Snowflake will auto-detect column types. Repeat for each CSV your problem requires.
SELECT * FROM HACKATHON.DATA.HR_ATTRITION LIMIT 5;
-- Replace table name to verify each CSV
3
Create your Streamlit app
Go to Projects → Streamlit → + Streamlit App. Select database HACKATHON, schema DATA, and your warehouse. Paste the Starter Code from your problem, then click Run to preview.
Name: your_app_name
Database: HACKATHON
Schema: DATA
Warehouse: <your warehouse>

Submission Rules
Guidelines & platform constraints

Keep it Snowflake-native. Everything runs inside your trial account — no external compute, no third-party LLMs. All work must be created during the 24-hour window.

Submission requirements

  • All code committed to a public GitHub repo before the deadline
  • README with setup instructions, architecture diagram, and demo link required
  • All models and notebooks must run in the provided Snowflake trial account
  • 3–5 minute video walkthrough is mandatory for all submissions

Platform constraints

  • All ML training and inference must occur within Snowflake
  • External LLM APIs (OpenAI, Anthropic, etc.) are not permitted — Cortex only

  • Analytics track: data must come from the provided CSV files only
  • Streamlit in Snowflake is the required UI layer for all deliverables
  • Open-source libraries permitted with attribution in the README
